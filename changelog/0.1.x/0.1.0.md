# 0.1.0 — 2026-06-24

Initial release. Human population genetics over the gnomAD GraphQL API for variant interpretation — keyless, single-source for the core surface, with ClinVar significance joined per variant.

## Added

- **`gnomad_get_variant`** — full population record for up to 25 variants per call (AC/AN/AF overall and per genetic-ancestry group, homozygote/hemizygote counts, quality flags, transcript consequence, in-silico predictors, joined ClinVar significance) with per-item partial success.
- **`gnomad_get_gene_constraint`** — gene loss-of-function constraint (pLI, LOEUF / `oe_lof_upper` with CI, observed/expected ratios, Z-scores) by HGNC symbol or Ensembl gene ID.
- **`gnomad_list_gene_variants`** — variants across a gene / transcript / region, filterable by consequence class and max AF; spills the full set to the `gene_variants` DataCanvas table.
- **`gnomad_get_coverage`** — per-track sequencing coverage (mean/median depth, fraction-of-samples-over-threshold) to distinguish a true absent variant from an uncallable position.
- **`gnomad_search_clinvar`** — gene-level ClinVar detail via NCBI E-utilities; spills to the `clinvar_variants` DataCanvas table. Honors an optional `NCBI_API_KEY`.
- **`gnomad_dataframe_query` / `gnomad_dataframe_describe`** — read-only SQL `SELECT` and table introspection over staged canvas tables.
- **`gnomad_dataframe_drop`** — opt-in canvas table drop, gated behind `GNOMAD_DATAFRAME_DROP_ENABLED` (off by default; absent from `tools/list` when off).
- **Resources** — `gnomad://variant/{dataset}/{variantId}` and `gnomad://gene/{dataset}/{gene}/constraint` mirroring the two scalar tools.
- **Prompt** — `gnomad_variant_triage`, the frequency → constraint → callability triage chain.
- **`dataset` + `reference_genome`** as distinct, coherence-validated parameters on every tool, echoed in output; per-ancestry frequency vector returned in full.
