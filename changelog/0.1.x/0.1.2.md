# 0.1.2 — 2026-06-27

## Fixed

- **`gnomad_search_clinvar`** — the `clinical_significance` filter is now enforced post-fetch. NCBI's `[clinical_significance]` field tag matches broadly, so a `pathogenic` query leaked `Benign`, `Uncertain significance`, and `Conflicting…` rows. Matching is whole-word and case-insensitive — `pathogenic` keeps `Pathogenic`, `Likely pathogenic`, and `Pathogenic/Likely pathogenic` but not `Conflicting classifications of pathogenicity`, and underscores read as spaces so the documented `likely_pathogenic` form works. The filter composes with `min_review_stars`; the prior early return applied only one when both were set. [#2](https://github.com/cyanheads/gnomad-genetics-mcp-server/issues/2)
- **`gnomad_list_gene_variants`** — `exome|genome` rows now report joint allele number (sum of carried-callset ANs, previously the max) and joint allele frequency (recomputed from joint `ac/an`, previously the single-callset upstream `af`). `max_af` now filters on the joint frequency the row exposes. Single-callset rows are unaffected; the `an`/`af` field descriptions are updated to match. [#3](https://github.com/cyanheads/gnomad-genetics-mcp-server/issues/3)
- **`gnomad_get_gene_constraint`** — the `content[]` LOEUF interval upper bound now renders `oe_lof_upper` instead of the `oe_lof` point estimate, which had contradicted the headline `oe_lof_upper` value; `oe_lof` is rendered explicitly as the point estimate on the same line. [#5](https://github.com/cyanheads/gnomad-genetics-mcp-server/issues/5)
